purpose: Validate if the Packaging Tool enable works well with the 'app-version', 'app-local-path', 'name', 'package', 'description', 'fullscreen', default 'icon', 'mode=embedded', 'orientation=landscape', 'target-dir' and default 'arch' options can package app on Windows host with absolute path

Test steps:

  1. Cd into packaging tool directory in Command Prompt. Eg: "cd C:\working\Crosswalk\Crosswalk-3.32.46.0\crosswalk"
  2. Package an web app with app-version option, command like: python make_apk.py --name=xxxx --package=xxxx.xwalk.org --app-root=app_path --app-local-path=index.html --app-version=1.0.0 --description=YourApplicationDescription --fullscreen --mode=embedded --orientation=landscape --target-dir=/home/path/to/
  3. Check the webapp version
  4. Check the packaging tool folder
  5. Check the value of "android:description" in app folder/AndroidManifest.xml
  6. Check the description in variable path (mostly in app folder/res/values/string.xml)
  7. Install the apk packaged
  8. Launch the webapp

Expected Output:

  1. There is a “make_apk.py” file in the folder
  2. Two apks created with "x86" and "arm" arch and embedded mode in the target dir correctly
  3. The version of the webapp is 1.0.0
  4. There generated an app folder
  5. There is variable path for description
  6. The description of the webapp is: YourApplicationDescription
  7. Web app could be installed successfully
  8. Web app could be launched with fullscreen and the screen orientation is landscape
