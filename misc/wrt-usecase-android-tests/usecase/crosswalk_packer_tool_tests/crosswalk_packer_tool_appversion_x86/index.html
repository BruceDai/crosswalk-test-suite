purpose: Validate if the Packaging Tool works well with the 'x86' arch, 'app-url', 'mode=shared', 'verbose' and 'keep-screen-on' options can package app with url on Windows host

Test steps:

  1. Cd into packaging tool directory in Command Prompt. Eg: "cd C:\working\Crosswalk\Crosswalk-3.32.46.0\crosswalk"
  2. Package an web app with 'x86' arch, command like: python make_apk.py --name=xxxx --package=xxxx.xwalk.org --app-url=http://www.intel.com --arch=x86 --app-version=1.0.0 --keep-screen-on --mode=shared
  3. Package the web app again with verbose option: python make_apk.py --name=xxxx --package=xxxx.xwalk.org --app-url=http://www.intel.com --arch=x86 --app-version=1.0.0 --keep-screen-on --mode=shared --verbose
  4. Compare the two packaging process messages in terminal
  5. Install the apk packaged
  6. Set the device screen off as 15 seconds(Setting->Display->Sleep: 15 seconds)
  7. Check the webapp version in setting
  8. Launch the webapp
  9. Check the screen

Expected Output:

  1. There is a “make_apk.py” file in the folder
  2. Apk with "x86" arch and shared mode could be created correctly
  3. Apk could be created correctly
  4. There got more detailed messages in the second packaging
  5. Web app could be installed successfully
  6. The version of the webapp is like 1.0.0.x
  7. Web app could be launched successfully, the home page display intel and keep screen on status more than 15 seconds
