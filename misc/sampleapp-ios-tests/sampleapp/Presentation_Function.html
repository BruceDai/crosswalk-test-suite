Test steps:

  1. Run below commands to build project under HexGL-iOS/HexGLAppp directory:
    $ open HexGLApp.xcodeproj
  2. Select the project "HexGLApp" --> set the active scheme --> HexGLAppp --> <iOS Device> -- > Product --> Build --> Run
  3. Check the test device
  4. Click "Hardware" --> "External Display" --> <Choose one value(such as 1024 x 768)>(if the test device isn't emulator, please skip this step)
  5. Change the "REMOTE DISPLAY:OFF" to "REMOTE DISPLAY:ON" on the lauching screen
  6. Click "Start"
  7. Touch the screen
  8. Play the game until display "DESTROYED"
  9. Touch the screen to restart the game

Expected Results:

  1. Open the app named "Xcode" and display one project named "HexGLApp" on the left
  2. Display message "Build Succeeded" and running the build app
  3. Launch the app named like "HexGL" successfully, and no error message, no crash, and display "START", "CONTAOLS", "REMOTE DISPLAY:OFF" and so on
  4. Open the External Display
  5. Display "REMOTE DISPLAY:ON"
  6. Display "CLICK/TOUCH TO CONTINUE", "TURN" and "ACCEL"
  7. Display the game main page normally on the external display and enter the "START" page and you can play the game when display "GO" on the screen
  8. Display message like: "CLICK/TOUCH TO CONTINUE" and play time
  9. Display "START", "CONTAOLS", "REMOTE DISPLAY:OFF" and so on
