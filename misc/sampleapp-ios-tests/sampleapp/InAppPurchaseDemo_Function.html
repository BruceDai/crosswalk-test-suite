Notice

The InAppPurchaseDemo require a specific Apple ID which is provided by the developers.
If you have your own Apple ID signed in an iOS device, please sign out before you start the tests.

Test steps:

  1. Build the project, please refer to Build InAppPurchaseDemo project
  2. Connect the iOS device to the Mac OS X system. In Xcode, select Product --> Run to launch the app on the iOS device
  3. Check the UI of app InAppPurchaseDemo, follow the text if there is any
  4. Purchae products:
    1. Click Query Products button
    2. Click Buy Apple button
    3. Select Use Existing Apple ID
    4. Input the Apple ID and password. For the first time it may require you to input twice. Then click OK button
    5. Click Confirm button
    6. Click Allow button
  5. Click Get Receipt button
  6. Input the password, and click OK button
  7. Click Validate Receipt button
  8. Click Restore button

Expected Results:

  1. Display message Build Succeeded and running the built app
  2. Launch the app named InAppPurchaseDemo successfully, and no error message, no crash
  3. There are 4 buttons Query Products, Validate Receipt, Get Receipt and Restore. There is some text below the buttons. The last line displays Initialization succeeded and the text contains the Apple ID account information for the test.
  4. Purchae products:
    1. Some virtual products like Cherry, Grape, Apple, Pear, Banana with price are listed
    2. A dialog popups to ask for choosing existing or new Apple ID
    3. A dialog popups to ask for the Apple ID account and password
    4. A dialog popups to ask to Confirm Subscription
    5. A dialog popups to ask to Allow to share the customer information
    6. The UI shows Purchase succeeded. There should be no text like "Operation failed"
  5. A dialog popups to ask for the password
  6. A long unreadable string shows behind the text Receipt. There should be no text like "Operation failed"
  7. The last line of the text should be Validation succeeded.
  8. Some product IDs list behind the text Products restored. There should be no text like "Restore failed"
