Test Steps

  1. Open one terminal command line window run terminal command, run one of the commands below:
    • $ python make_apk.py --name=<name> --package=<package> --app-url=http://www.baidu.com --arch=x86
    • $ python make_apk.py --package=<package> --manifest=<manifest-path> --arch=arm
    • $ python make_apk.py --app-local-path=index.html --app-root=<app-root-path>
  2. Run below commands in another terminal command window during the step 1 is running in first window
    $ cd /tmp
    $ ls (you may need to repeat this command serveral times)
  3. Install the packaged apk
  4. Launch the web app
  5. Repeat steps 1-4 with another commands of step 1
  6. Run the command:
    $ python make_apk.py --package=<package> --manifest=<manifest-path> --project-dir=<dest_dir>
  7. Run the command:
    $ cd <dest_dir>/<project_name>
    Then check the folder <project_name>
  8. Run the command:
    $ ant release -f build.xml
  9. Check the apk packages exists in bin/
  10. Install the apk <project_name>-release-unaligned.apk
  11. Launch the web app
  12. Run the command:
    $ python make_apk.py --package=<package> --manifest=<manifest-path> --project-only
  13. Run the command:
    $ python make_apk.py --package=<package> --manifest=<manifest-path> --project-only --project-dir=<dest_dir>

Purpose:

  • Verifies the nothing changes in the current working directory during a build
  • Verifies a working project directory can be created during the build for use later to create valid APKs
  • Verifies APK creation can be skipped if developer only wants to create a project directory for use later

      • Expected Result:

      1. Create web app successfully
      2. Display the created app project folder
        For example: --package=org.hello.tmptest, the folder in /tmp should be "Tmptest"
      3. Install the app successfully
      4. Launch the app successfully
      5. Run another two times successfully
      6. Create one new project after the build in <dest_dir>/<project_name>
      7. Display some set of key files, such as assets, bin, src and AndroidManifest.xml and so on
      8. Build and exit successfully, no error message
      9. Display <project_name>-release*.apk
      10. Install the app successfully
      11. Launch the app successfully
      12. Display the error message like '--project-only must be used with --project-dir'
      13. Display the folder <dest_dir>/<project_name>, no create apk package
