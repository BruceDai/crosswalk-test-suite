Preconditions

  1. Requires the latest Cordova CLI, and must >= 5.0.0, install with command: '$ sudo npm install cordova -g'

Test Steps

  1. Get cordova-plugin-crosswalk-webview:
    $ git clone https://github.com/crosswalk-project/cordova-plugin-crosswalk-webview.git
  2. Create Cordova example app, e.g. “hello”:
    $ cordova create hello com.example.hello HelloWorld
  3. Setup Cordova example app:
    $ cd hello
    $ cordova platform add android
    $ sed -i "s/<\/widget>/    <preference name=\"Fullscreen\" value=\"true\" \/>\\n<\/widget>/g" config.xml
    $ sed -i "s/<\/widget>/    <preference name=\"disableImmersive\" value=\"true\" \/>\\n<\/widget>/g" config.xml
    $ cordova plugin add --plugin org.apache.cordova.statusbar
    $ cordova plugin add ../cordova-plugin-crosswalk-webview
  4. Copy the 'usecase-cordova-android-tests/res/StatusBar/index.html' file to hello/www directory
  5. Build Cordova example app:
    $ cordova build android
  6. Check the apk file generated in hello/build/outputs/apk directory
  7. Install, Launch, Run, Click the buttons and Exit the app

Purpose:

Verify the Cordova StatusBar plugin can work when app in fullscreen mode

Expected Results:

  1. Create test suite(hello) successfully
  2. Build the apk successfully
  3. The app can be installed successfully
  4. The app can launch successfully
  5. The app can run successfully
  6. The app show the StatusBar after click the "Status Bar Show" button
  7. The app hide the StatusBar after click the "Status Bar Hide" button
  8. The app can exit successfully
